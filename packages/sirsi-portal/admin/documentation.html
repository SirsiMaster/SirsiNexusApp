Admin Console Documentation

Welcome to the SirsiNexus Admin Console documentation. This guide will help you understand and effectively use all features of the admin portal.

Ctrl+K

Quick Start

  1. Log in to the admin console with your administrator credentials
  2. Navigate through the sidebar to access different sections
  3. Use Quick Actions for common tasks
  4. Monitor system health from the dashboard
  5. Configure settings as needed for your organization

Dashboard Guide

Overview

The dashboard provides a real-time overview of your platform's performance, user activity, and system health.

Key Metrics:

  • Total Users: Active user count and growth percentage
  • Revenue: Monthly revenue tracking and trends
  • Active Sessions: Real-time user sessions
  • Server Health: System uptime and performance

Quick Actions Available:

Add User: Create new user accounts
Send Invite: Invite users via email
Generate QR: Create access QR codes
Role Management: Configure user permissions

User Management

Managing Users

The User Management section allows you to create, edit, and manage user accounts and permissions.

Key Features:

  • User Creation: Add new users with role assignment
  • Bulk Import: Import multiple users via CSV
  • Role Management: Define custom roles and permissions
  • Activity Tracking: Monitor user login and actions
  • Access Control: Set department and data access levels

User Roles:

Admin - Full system access Manager - Department management access Analyst - Read-only data access Viewer - Limited viewing permissions

Data Room Management

Document Management

Securely manage and share confidential documents with authorized users.

Features:

  • Secure Upload: Encrypted file storage
  • Access Control: Granular permission settings
  • Version Control: Track document revisions
  • Audit Trail: Monitor document access and downloads
  • Watermarking: Automatic document watermarking

Supported File Types:

PDF, DOCX, XLSX, PPTX, Images (JPG, PNG), and more

Monitor real-time system performance and user behavior analytics.

Available Metrics:

Performance Metrics:

User Analytics:

Export Options:

Export data as CSV, PDF reports, or integrate with analytics platforms via API.

Security Configuration

Security Management

Security Features:

  • Access Control: IP whitelisting and blacklisting

Important: Always test security changes in a staging environment before applying to production.

System Logs

Log Management

Log Types:

  • Access Logs: User login and page access
  • Error Logs: System errors and exceptions
  • Security Logs: Security events and alerts
  • API Logs: API request and response data

Log Filtering:

Filter logs by date range, severity level, user, or event type.

Example filters: - Date: Last 7 days - Severity: Error, Warning - User: admin@example.com - Event: login_failed

API Reference

Admin API Endpoints

Authentication:

POST /api/auth/login Authorization: Bearer {token}

Common Endpoints:

GET /api/users - List all users POST /api/users - Create new user GET /api/users/{id} - Get user details PUT /api/users/{id} - Update user DELETE /api/users/{id} - Delete user GET /api/logs - Retrieve system logs GET /api/metrics - Get platform metrics POST /api/settings - Update settings
