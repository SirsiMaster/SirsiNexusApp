## 🔧 Architecture Summary

### The Dual-Shard Architecture (Patent Core)

Sirsi's distributed intelligence operates on two parallel planes:

1. **Tensor Shards (Compute Layer)** — Distribute ML inference across multiple silicon nodes. When a model exceeds a single node's memory, weights are partitioned across nodes with high-speed interconnect for intermediate activations.

2. **Logic Shards (Consensus Layer)** — Consensus-ordered reasoning coordination. Every AI decision, infrastructure state verification, and agent coordination message is deterministically ordered, immutably recorded, and cryptographically verifiable.

**Together:** The compute layer handles *how* to compute. The consensus layer handles *what*, *who*, *when*, and *why* — with mathematical proof.

### Knowledge Graph + Truth Anchoring

The platform's Knowledge Graph models all infrastructure as a directed graph (nodes = services/hardware/configs, edges = relationships). Every KG state is hashed and anchored on a distributed ledger, creating immutable proof of infrastructure state at any point in time.
